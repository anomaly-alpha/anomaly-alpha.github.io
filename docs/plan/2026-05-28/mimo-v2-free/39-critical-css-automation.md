# Plan 39: Automated Critical CSS Extraction

**Problem:** Critical CSS is manually maintained as inline `<style>` blocks in every page. When new Tailwind classes are added to the HTML, the inline critical CSS must be manually updated. This has caused CLS issues before (as documented in the Lighthouse audit).

**Goal:** Automate critical CSS extraction so it stays in sync with the HTML. Run it as part of the build pipeline.

---

## Step 1: Install critical CSS tool

```bash
npm install -D critical
```

---

## Step 2: Create extraction script

**File: `scripts/extract-critical.js`**

```js
const critical = require('critical');
const path = require('path');
const fs = require('fs');

const PAGES = [
  { src: 'index.html', target: 'index.html', width: 1300, height: 900 },
  { src: 'guide/code/index.html', target: 'guide/code/index.html', width: 1300, height: 900 },
  { src: 'guide/event/index.html', target: 'guide/event/index.html', width: 1300, height: 900 },
  { src: 'guide/pvp/index.html', target: 'guide/pvp/index.html', width: 1300, height: 900 },
  { src: 'guide/login/index.html', target: 'guide/login/index.html', width: 1300, height: 900 },
  { src: 'guide/faq/index.html', target: 'guide/faq/index.html', width: 1300, height: 900 },
  { src: 'guide/beginners/index.html', target: 'guide/beginners/index.html', width: 1300, height: 900 },
  { src: '404.html', target: '404.html', width: 1300, height: 900 },
];

async function extractAll() {
  for (const page of PAGES) {
    console.log('Extracting critical CSS for:', page.src);

    try {
      const result = await critical.generate({
        inline: true,
        base: path.join(__dirname, '..'),
        src: page.src,
        target: page.target,
        width: page.width,
        height: page.height,
        minify: true,
        ignore: {
          atrule: ['@font-face'],
          rule: [/.gem-particle/, /.gem-orb/]
        }
      });

      console.log('  ✓ Inlined into', page.target);
    } catch (err) {
      console.error('  ✗ Failed:', err.message);
    }
  }
}

extractAll();
```

---

## Step 3: Add npm script

```json
"extract-critical": "node scripts/extract-critical.js"
```

And integrate into build:

```json
"build": "npm run build:tailwind && npm run build:css && npm run build:js && npm run extract-critical"
```

---

## Step 4: Update the build pipeline (Plan 01/07)

After Tailwind generates `tailwind.css` and csso minifies it, run `extract-critical` to:
1. Load each HTML page
2. Render it in a headless browser at 1300×900
3. Extract the CSS used above the fold
4. Inline it into a `<style>` block in the `<head>`
5. Keep full CSS as async

---

## Step 5: Remove manual critical CSS blocks

After extraction is working, the manually maintained critical CSS in each page can be replaced:

```html
<!-- Before: manually maintained critical CSS -->
<style>
html { visibility: hidden }
... 50+ lines of Tailwind rules ...
html { visibility: visible }
</style>

<!-- After: auto-generated by extract-critical -->
<!-- critical CSS will be inlined by the extract script -->
```
