# Plan 01: Source Maps for Production Debugging

**Problem:** When `npm run build` minifies script.js via terser, debugging production bugs is nearly impossible since the stack trace points to line/column in the minified output rather than the original source.

**Goal:** Generate a source map alongside minified script.js so Chrome DevTools can map minified positions back to the original content. Verified by opening DevTools Sources and seeing original variable names.

---

## Step 1: Add source-map-loader and configure terser
Install `source-map-loader` dev dependency and update the build command.

```bash
npm install --save-dev source-map-loader
```

In `package.json`, add `TERSER_SOURCEMAP=true` to the build script, or configure terser inline in the build script. Since terser CLI does not directly output source maps, replace the terser invocation in `package.json` with a small node wrapper or use `esbuild` for minification (which has built-in sourcemap support).

The simplest path: replace terser with esbuild for JS minification:

```json
{
  "scripts": {
    "build": "npm run build:tailwind && npm run minify:css && npm run minify:js",
    "minify:js": "esbuild script.js --minify --sourcemap --outfile=script.min.js"
  }
}
```

```bash
npm install --save-dev esbuild
```

## Step 2: Update HTML to reference minified + source map
Update `index.html` and all guide pages to reference `script.min.js` instead of `script.js`. The source map `script.min.js.map` is auto-generated by esbuild alongside it.

```html
<!-- In all HTML files, change: -->
<!-- <script src="script.js"></script> -->
<script src="script.min.js"></script>
```

Also add a comment at the bottom of the source map file to prevent accidental exposure:

```json
// script.min.js.map
{"version":3,"sources":["../script.js"],...}
```

## Files Modified
- `package.json` — add esbuild, update scripts
- `script.js` — already exists, no changes needed
- `index.html` — update script src
- `guide/*/index.html` — update script src

## Verification
```bash
npm run build
ls -la script.min.js script.min.js.map
# Open Chrome DevTools > Sources, confirm script.js appears with original variable names
```
